HTML

  • Hyper Text Markup Language or HTML is the Framework of an application
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Order matters. Be sure to to keep your header elements in order and only use 1 "h1" per webpage
  • Unordered lists are created using the "ul" element, which then contains several "li" elements to create a bulleted list.

CSS

  • Cascading Style Sheet or CSS is used to change the style of your application
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • when adding a style to the "*" attribute, you are making the change to all attributes.

Git

  • Git and GitHub are used to track changes within code and allows devs to make multiple changes at the same time without compromising the original code.
  • git status: checks what branch we are currently on
  • git checkout branch-name: switches to whichever branch that you listed
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages changes in Github.
  • git commit -m "your message here": writes a message to help define the commit.
  • git pull origin main: syncs the local branch with the one on the cloud.

JavaScript

  • JavaScript is used to make an application dynamic and interactive.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • When using if statements, the "===" condition is a strict equality operator that returns a true boolean if 2 values are equal.
  • Arrays are represented in a linear list starting with 0 not 1.
  • Don't forget!: You can write a function in your .js file, but you still have to call it to run it.
